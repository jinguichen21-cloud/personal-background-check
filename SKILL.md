# 个人背景调查

个人背景调查助手，集成个人被执行信息查询、身份证实名认证等功能。

## 概述

本技能提供全面的个人背景调查能力：
- 查询个人被执行信息（法院执行案件）
- 查询失信被执行信息（老赖名单）
- 查询限制高消费信息
- 身份证实名认证（姓名+身份证号核验）
- 生成背景调查报告

**适用场景**：
- 入职前背景调查
- 合作方资质审查
- 投资/借贷风险评估
- 商务合作尽职调查

## 前置准备

本技能需要以下 MCP 服务：

| 环境变量 | MCP 服务 | mcpId | 说明 |
|----------|----------|-------|------|
| `$PERSONAL_ENFORCEMENT_URL` | 个人被执行信息查询 | 2489 | 查询个人被执行、失信、限高信息 |
| `$ID_VERIFY_URL` | 身份证实名认证 | 9366 | 身份核验（姓名+身份证号匹配） |

**配置说明**：
1. `mcp-config.json` 中已配置 mcpId，URL 字段由市场链路根据 mcpId 自动生成
2. 本地开发测试时，可手动填入从钉钉 MCP 市场获取的 StreamableHTTP URL
3. `mcp-config.json` 不会打包进 `.skill`，请勿提交到代码仓库

详细说明见 [mcp-server-config-guide.md](references/mcp-server-config-guide.md)

## 核心工作流程

### 场景 1：入职前背景调查

```
HR提供候选人信息（姓名+身份证号）
    ↓
身份证实名认证（确认信息真实性）
    ↓
查询个人被执行信息
    ↓
生成背景调查报告
    ↓
返回调查结果和风险提示
```

### 场景 2：合作方资质审查

```
用户提供合作方负责人信息
    ↓
身份证实名认证
    ↓
查询个人司法信息
    ↓
分析风险等级
    ↓
生成资质审查报告
```

## 详细工作流程

### 场景 1：入职前背景调查

**触发条件**：HR需要对候选人进行背景调查

**处理步骤**：

1. **收集候选人信息**
   - 姓名（必填）
   - 身份证号（必填）
   - 手机号（可选）

2. **身份证实名认证**
   - 调用 `verify_idcard_query`
   - 验证姓名和身份证号是否匹配
   - 确认信息真实性

3. **查询个人被执行信息**
   - 调用 `query_personal_enforcement`
   - 查询法院执行案件信息

4. **分析查询结果**
   - 是否有被执行记录
   - 执行标的金额
   - 案件状态
   - 执行法院

5. **生成背景调查报告**

6. **返回调查结果**
   ```
   📋 背景调查报告
   
   👤 候选人：张三
   🆔 身份证号：110101********1234
   
   ✅ 身份证实名认证：通过（姓名与身份证号一致）
   
   🔍 司法信息查询结果：
   ┌─────────────────┬──────────┐
   │ 查询项目        │ 结果     │
   ├─────────────────┼──────────┤
   │ 被执行信息      │ 无记录 ✅ │
   │ 失信信息        │ 无记录 ✅ │
   │ 限制高消费      │ 无记录 ✅ │
   └─────────────────┴──────────┘
   
   🎯 综合评估：低风险，建议录用
   
   📅 报告生成时间：2026-03-20
   ```

### 场景 2：合作方资质审查

**触发条件**：需要对合作方负责人进行资质审查

**处理步骤**：

1. 获取合作方负责人信息
2. 身份证实名认证
3. 查询个人司法信息
4. 分析风险等级：
   - **高风险**：有被执行记录且金额较大、失信被执行
   - **中风险**：有被执行记录但已结案、金额较小
   - **低风险**：无被执行记录
5. 生成资质审查报告

## 工具说明

### 个人被执行信息查询 (mcpId: 2489)

#### query_personal_enforcement
查询个人被执行详细信息。

**参数**：
- `name` (string, 必填): 姓名
- `idCard` (string, 必填): 身份证号
- `phone` (string, 可选): 手机号

**返回值**：
- `hasRecord`: 是否有被执行记录
- `records`: 被执行记录列表
  - `caseNo`: 案号
  - `courtName`: 法院名称
  - `caseStatus`: 案件状态
  - `executionTarget`: 执行标的（金额）
  - `executionStatus`: 执行情况
  - `publishDate`: 发布日期

**示例调用**：
```bash
python3 scripts/call_mcp.py call "$PERSONAL_ENFORCEMENT_URL" query_personal_enforcement \
  --params '{"name": "张三", "idCard": "110101199001011234"}'
```

### 身份证实名认证 (mcpId: 9366)

#### verify_idcard_query
身份证实名认证。根据姓名和身份证查询信息是否一致。

**参数**：
- `name` (string, 必填): 姓名
- `idCard` (string, 必填): 身份证号

**返回值**：
- `isMatch`: 是否匹配（一致/不一致）
- `verifyResult`: 验证结果详情
- `message`: 验证结果说明

**示例调用**：
```bash
python3 scripts/call_mcp.py call "$ID_VERIFY_URL" verify_idcard_query \
  --params '{"name": "张三", "idCard": "110101199001011234"}'
```

## 风险等级评估标准

| 风险等级 | 判定标准 | 建议措施 |
|----------|----------|----------|
| **🔴 高风险** | 有失信被执行记录 / 被执行金额 > 100万 / 多条被执行记录 | 建议谨慎合作/不予录用 |
| **🟡 中风险** | 有被执行记录但已结案 / 被执行金额 < 10万 / 单条记录 | 需进一步核实情况 |
| **🟢 低风险** | 无被执行记录 / 身份证实名认证通过且无司法风险 | 正常合作/建议录用 |

## 隐私与合规说明

### 数据使用规范

1. **合法用途**
   - 仅用于入职背景调查、合作方审查等合法场景
   - 不得用于非法用途或侵犯个人隐私

2. **信息保护**
   - 查询结果仅用于内部评估
   - 不得泄露给无关第三方
   - 妥善保管调查报告

3. **告知义务**
   - 建议告知被调查人将进行背景调查
   - 获得必要的授权同意

4. **数据留存**
   - 调查报告建议留存备查
   - 定期更新复查（建议每半年）

### 免责声明

- 查询结果仅供参考，不作为唯一决策依据
- 数据来源于公开渠道，可能存在滞后性
- 建议结合其他渠道信息进行综合评估

## 异常处理

- **查询无结果** → 可能是信息有误或无相关记录，建议核实信息后重试
- **身份证认证失败** → 姓名与身份证号不匹配，建议核实信息
- **服务异常** → 提示用户稍后重试或联系管理员
- **权限不足** → 检查 MCP 配置是否有权限访问该服务
